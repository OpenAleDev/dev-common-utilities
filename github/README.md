# dev-common-utilities

Common utilities for development — managed via Docker containers.

---

## `github/` — GitHub Account Manager

A Docker-based environment to manage a GitHub account from a Windows host machine.  
It provides a Linux container with `git` and SSH authentication pre-configured, with the local repositories folder mounted as a volume.

### Project structure

```
github/
├── config/
│   ├── config.json           # Your local config (gitignored)
│   └── config.example.json   # Template to copy from
├── docker/
│   ├── .env                  # Generated by github_up.ts (gitignored)
│   ├── Dockerfile
│   ├── docker-compose.yml
│   └── entrypoint.sh         # Runtime setup: SSH keys + git identity
├── .ssh/                     # Generated SSH key pair (gitignored)
│   ├── id_ed25519
│   └── id_ed25519.pub
├── github_up.ts              # Initialisation script (run once)
├── buildDockerWithoutCache.sh
└── openShellContainer.sh
```

---

### First-time setup

#### 1. Prerequisites

- [Bun](https://bun.sh) installed on the host
- Docker Desktop running

#### 2. Configure `config.json`

Copy the example and fill in your details:

```bash
cp config/config.example.json config/config.json
```

```json
{
    "github_account_email": "you@example.com",
    "github_account_username": "your-github-username",
    "github_repo_local_folder": "C:/path/to/your/repos"
}
```

#### 3. Run the init script

```bash
bun run github_up.ts
```

This script will:
- Generate `docker/.env` with `GITHUB_EMAIL`, `GITHUB_USERNAME`, `REPO_ROOT_PATH`, and `GITHUB_SSH_KEY_PATH`
- Generate an ed25519 SSH key pair in `.ssh/`
- Build and start the Docker container

> **The script is safe to re-run** — it checks whether `docker/.env` and `.ssh/` already exist and aborts with an error if they do, to avoid overwriting existing keys or config.

#### 4. Register the public key on GitHub

Copy the content of `.ssh/id_ed25519.pub` and add it to your GitHub account:

**GitHub → Settings → SSH and GPG keys → New SSH key**

#### 5. Make sure your remotes use SSH URLs

GitHub remotes must use the SSH format, not HTTPS:

```bash
# Check
git remote -v

# Fix if HTTPS
git remote set-url origin git@github.com:<username>/<repo>.git
```

### Daily usage

**Start the container:**

```bash
cd docker && docker compose up -d
```

**Open a shell inside the container:**

```bash
sh openShellContainer.sh
# equivalent to: docker exec -it github-repo-manager bash
```

**Rebuild from scratch (no cache):**

```bash
sh buildDockerWithoutCache.sh
# equivalent to: cd docker && docker compose up -d --build --no-cache
```

**Stop the container:**

```bash
cd docker && docker compose down
```

Inside the container, all your repositories are accessible at `/repos`.

---

### How it works

#### Environment variables
`docker-compose.yml` reads `docker/.env` and injects `GITHUB_EMAIL` and `GITHUB_USERNAME` as runtime environment variables into the container.

#### SSH keys
The `.ssh/` folder is mounted read-only into the container at `/root/.ssh-host` (staging path).  
At container startup, `entrypoint.sh` copies the keys into `/root/.ssh` and applies the correct permissions.

#### Git identity
`entrypoint.sh` runs on every container start and calls `git config --global` with the values from the environment variables.

### Known issues & solutions

#### ❌ `Author identity unknown` on commit
**Cause:** `git config` was set at Docker build time via `ARG`, but build cache or empty arg values prevented it from being written.  
**Fix:** Moved `git config` to `entrypoint.sh`, which runs at container startup and reads live environment variables.

---

#### ❌ `Permissions 0777 for '/root/.ssh/id_ed25519' are too open`
**Cause:** Docker on Windows assigns `0777` to all volume-mounted files. SSH refuses private keys that are world-readable. The permissions cannot be changed inside the container on a mounted volume.  
**Fix:** The `.ssh/` folder is mounted to a staging path (`/root/.ssh-host`) instead of directly to `/root/.ssh`. The entrypoint copies the files to `/root/.ssh` (a real container filesystem path) and applies `chmod 600`.

---

#### ❌ Git push asks for username and password
**Cause:** The repository remote URL uses HTTPS instead of SSH.  
**Fix:**
```bash
git remote set-url origin git@github.com:<username>/<repo>.git
```

---

#### ❌ `Failed to add the host to the list of known hosts`
**Cause:** `/root/.ssh/known_hosts` was not present, causing SSH to prompt interactively (and fail in non-interactive contexts).  
**Fix:** `entrypoint.sh` runs `ssh-keyscan -H github.com` on every startup to keep `known_hosts` populated.

---

#### ❌ `entrypoint.sh` silently does nothing (script not executed)
**Cause:** The file was created on Windows with CRLF line endings. Linux bash cannot parse the `#!/bin/bash\r` shebang.  
**Fix:** `dos2unix` is installed in the image and run on `entrypoint.sh` at build time (`RUN dos2unix /entrypoint.sh`).
